# Workload Suite Schema

Suites are small manifest-defined groups of prompts run through `scripts/run_experiment.ps1`.

## Required sections
- `name`: suite name used in `runs/suites/<name>/<timestamp>`.
- `paths.model`: GGUF model path.
- `paths.runtime`: `llama-server.exe` path.
- `paths.experiment_runner`: path to `scripts/run_experiment.ps1`.
- `server`: frozen server flags for the suite.
- `generation`: fixed generation settings for all cases.
- `output.suite_root`: root directory for suite artifacts.
- `case_<id>`: one section per workload case.

## Case fields
- `order`: numeric report/run order.
- `label`: short display label.
- `prompt_file`: prompt file passed to `run_experiment.ps1`.

## Required artifacts
- `suite_result.json`: suite metadata and child run folders.
- `report.md`: compact metrics table generated by `scripts/report_suite.ps1`.
- child run folders: one normal experiment run per case.

## Report fields
- `prompt_n`
- `predicted_n`
- `ttft_est_ms`
- `wall_ms`
- `decode_tps`
- `coherent`
- `cpu_mapped_model_buffer`
- `cuda_model_buffer`
- `kv.seq_rm_capability`
- `kv.kv_policy_status`
